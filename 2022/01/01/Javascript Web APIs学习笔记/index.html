Javascript Web APIs简介

API - Application programming interface 应用程序编程接口, 是一些预先定义的函数, 目的是为应用程序和开发人员提供一些功能, 而不需要去访问或理解源码, 或是接口内部的工作机制.

Web API - 是浏览器提供的一套操作浏览器功能和页面元素的API (也就是我们熟知的BOM 和 DOM). Web APIs是由W3C组织指定的标准.

一些常用的 Web APIs 可以参考: https://developer.mozilla.org/en-US/docs/Web/API

DOM

Document Object Model, 又称文件对象模型. 是W3C组织推荐的处理 HTML 的标准编程接口. 通过使用 DOM接口, 我们可以改变网页的内容, 结构和样式.

DOM树

如果把一个页面中的HTML元素标签和元素属性看作是一个个的节点, 他们组成的图形就象是一棵树.

DOM树包含的以下节点:

  • 文档 Document: 一个页面就是一个文档

  • 元素 Element: 页面中的所有标签都是元素

  • 节点 Node: 页面中所有的内容都是节点 (包括标签节点, 属性节点, 文本节点, 注释节点等等)

事件

JavaScript 使我们有能力创建动态页面。事件是可以被 JavaScript 侦测到的行为。 网页中的每个元素都可以产生某些可以触发JavaScript函数的事件。比方说，我们可以在用户点击某按钮时产生一个 onClick 事件来触发某个函数。事件在 HTML 页面中定义。

事件三要素

即组成事件的三部分分别为:

  1. 事件源: 事件被触发的元素对象.
  2. 事件类型: 事件是如何被触发的. 比如鼠标点击 onclick 或 失去鼠标焦点 onBlur.
  3. 事件处理程序: 事件触发后会运行的 JS 代码. 比如弹出对话框, 改变元素的样式等等.

事件执行的步骤

  1. 获取事件源 (通过之前介绍的元素获取方法)
  1. 注册(绑定)事件, 以下是以下常见的鼠标事件:

onmouseover 鼠标经过
onmouseout 鼠标移出
onfocus 获得鼠标焦点
onblur 失去鼠标焦点
onmousemove 鼠标移动
onmouseup 鼠标弹起
onmousedown 鼠标按下

修改元素的样式

通过element.style来修改样式

修改元素样式可以通过调用元素对象的 style属性来完成. 该属性下的样式名都使用了驼峰命名 (比如在外部样式标签中, 背景颜色为 background-color, 但是背景应是 backgroundColor).

注意: 通过这种方法修改的样式, 等同于给元素添加了行内样式 (行内样式优先级高于内部/外部样式, 所以两者有冲突时, 会优先使用行内样式).

通过改变类名来修改样式

如果要修改的样式较多, 我们可以把这些样式写在css文件中, 并用选择器选择特定类名, 最后给元素对象添加这个类名, 就可以以此改变元素的样式.

操作元素对象自定义属性值

  • 获取自定义属性值

方法1: 之前我们使用了 element.属性名的方法来获取属性值 (主要用来获元素自带的属性的值). 另一种获取属性值方法是 element.getAttribute(属性名). (主要用来获取程序员自定义的元素属性的值).

方法2: 通过element.dataset.属性名/ element.dataset[属性名], 使用这种方法的前提是自定义属性命名时必须遵守 HTML5 的规定, 即 data-属性名. (元素对象的dataset属性中, 会存放所有以 data-为开头的属性)

  • 修改自定义属性值

如果要改变自定义元素属性的值, 我们需要使用方法: element.setAttribute(属性名, 值)

  • 删除自定义属性值

如果要删除一个自定义属性, 我们需要使用方法: element.removeAttribute(属性名)

自定义属性

作用: 保存一些数据, 这样我们可以在不访问数据库的情况下, 保存和提取某些数据.

规范: HTML5规定了自定义属性的命名规范, 即必须为data-属性名的结构. (所以如果你看到html标签有以data为开头的属性, 这一定是一个自定义属性).

节点操作

之前我们学习了怎么使用DOM提供的方法来获取元素对象, 较为繁琐. 接下来我们要学习怎样利用节点的层级关系来获取元素对象.

节点概述

节点是HTML元素标签和元素属性的抽象. 每个节点都至少有用三个基本属性: nodeType, nodeName 和 nodeValue. 即节点类型, 节点名, 节点值.

nodeType 节点类型 - 元素节点的节点类型为1, 属性节点则为2, 文本节点则为3.

节点的层级关系

依照DOM树, 我们可以把所有节点分化为不同的层级关系. 以DOM树为例, 存在包含关系的节点, 比如根节点 <html> 的子节点包括<head>和<body>, 他们的关系就是父子关系. 而<head> 和<body>就是兄弟关系.

  • 父节点 - 我们可以通过调用子节点的element.parentNode来访问其父节点. 如果找不到父节点, 则该属性值为null.

  • 子节点 - 我们可以通过调用父节点的element.childNodes来访问其所有的子节点 (包括元素节点, 属性节点, 文本节点等等). 开发一般我们只关心元素节点, 所以我们需要通过nodeType==1来确定一个子节点是否是元素节点, 再进行后续操作. 另外一种方法是调用父节点的children属性, 这个属性只包含元素子节点, 要注意的是这个属性虽然被大部分浏览器支持, 但并不符合W3C的标准.
